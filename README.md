

# 🤖 BidMate: AI RFP 분석기 (RAG System)

**BidMate**는 입찰 제안요청서(RFP)를 분석하고, 궁금한 점을 AI에게 질의할 수 있는 **RAG(검색 증강 생성) 기반 챗봇 서비스**입니다.  
HWP, PDF 등 다양한 형식의 문서를 자동으로 읽어 벡터 DB를 구축하고, 사전에 구성된 로컬 LLM 서버와 연동하여 안전하게 분석합니다.

---

## ✨ 핵심 기능 (Key Features)

* **📄 HWP & PDF 완벽 지원**: 표나 특수 서식이 많은 공공기관 문서(HWP)의 텍스트도 정확하게 추출합니다.
* **🧠 듀얼 검색 엔진**:
    * `bge-m3`: 다국어 및 범용 검색용 (기본)
    * `kure-v1`: 한국어 특화 경량 모델
* **🤖 실시간 모델 변경**:
    * `Gemma 3 (12B)`: 심층 분석 및 논리적 답변
    * `Llama 3.1`: 빠른 응답 속도
* **🔍 투명한 출처 표기**: 답변의 근거가 된 문서 이름과 내용을 명확히 제시합니다.

---

## 🚀 빠른 시작 (Quick Start)

이미 AI 모델 서버가 준비되어 있으므로, 아래 명령어 하나만 입력하면 바로 실행됩니다.

### 1. 프로젝트 실행 (권장)
초기 셋업(가상환경, 라이브러리)부터 데이터 연결, DB 생성, 앱 실행까지 한 번에 수행합니다.

```bash
make run

```

### 2. 앱 재실행

이미 설치가 완료된 상태에서 웹 앱만 다시 켜고 싶을 때는:

```bash
make app

```

---

## 📂 수동 설정 가이드 (Manual Setup)

`make` 명령어를 사용하지 않고 단계별로 진행하고 싶을 때 참조하세요.

1. **데이터 연결**: `make link_data` (서버의 공유 데이터를 프로젝트로 연결)
2. **가상환경 세팅**: `make setup` (Python 가상환경 생성 및 패키지 설치)
3. **벡터 DB 구축**: `make db` (문서를 읽어 검색용 DB 생성)
4. **앱 실행**: `streamlt run app.py`

## ⚠️ 트러블슈팅 (Troubleshooting)

**Q. "Connection refused" 에러가 발생해요.**

> **A.** 백그라운드 AI 서버(Ollama)가 켜져 있는지 확인해주세요. (기본 포트: 11434)

**Q. "처리할 문서가 없습니다"라고 뜹니다.**

> **A.** 데이터 연결이 끊어진 경우입니다. `make clean` 후 `make run`을 다시 실행하여 연결을 복구하세요.
